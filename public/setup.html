Подключение Google и Яндекс OAuth

Чтобы пользователи могли входить через соцсети, создайте OAuth-приложения и добавьте ключи в файл `.env`.

Статус настройки:

Google OAuth: Проверка...

Яндекс OAuth: Проверка...

1 Установите Node.js

Скачайте и установите с nodejs.org (версия LTS). После установки перезапустите компьютер.

2 Запустите сервер

Дважды щёлкните start.bat в папке проекта или выполните в терминале:

npm install
npm start

Сайт должен открываться по адресу http://localhost:3000 (не через файл index.html).

3 Создайте OAuth в Google Cloud

  1. Откройте Google Cloud Console
  2. Создайте OAuth 2.0 Client ID
  3. Authorized redirect URIs: http://localhost:3000/auth/google/callback

В .env:

GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...

4 Создайте OAuth в Яндекс ID

  1. Откройте oauth.yandex.ru
  2. Создать приложение
  3. Платформы: Веб-сервисы
  4. Redirect URI: http://localhost:3000/auth/yandex/callback
  5. Доступы: Яндекс ID (Почта, Имя, Фамилия, Аватар)

В .env:

YANDEX_CLIENT_ID=...
YANDEX_CLIENT_SECRET=...

5 Перезапустите сервер

Остановите сервер (Ctrl+C) и снова запустите npm start или start.bat.

После этого кнопка «Войти через Google» заработает. Первый вошедший пользователь автоматически станет администратором.

6 amoCRM (внешняя CRM)

Каждый заказ автоматически создаёт сделку в amoCRM с контактом клиента и составом заказа.

  1. Зарегистрируйтесь на amocrm.ru
  2. Откройте amoМаркет → Создать интеграцию → Внешняя интеграция
  3. Скопируйте долгосрочный токен (Access token)
  4. Поддомен — часть адреса: https://company.amocrm.ru → company

Добавьте в .env или Render → Environment:

CRM_PROVIDER=amocrm
AMOCRM_SUBDOMAIN=company
AMOCRM_ACCESS_TOKEN=ваш-токен
AMOCRM_PIPELINE_ID=123456

AMOCRM_PIPELINE_ID — ID воронки (необязательно). Найти: Настройки → Воронки → в URL будет /pipeline/123456

Админ магазина: меню «CRM» → откроется воронка ScentForge (только «Новый заказ», без лишних этапов).

7 ИИ-помощник (опционально, бесплатно)

Для полноценного ИИ с пониманием любых формулировок:

  1. Откройте Google AI Studio
  2. Нажмите Create API Key
  3. Добавьте в .env: GEMINI_API_KEY=ваш-ключ
  4. Перезапустите сервер

Без ключа работает умный подбор по смыслу запроса.

8 Письма о заказах (SMTP)

После заказа письмо приходит на Google-почту клиента (через которую он вошёл).

Вариант А — Gmail (если доступны пароли приложений)

  1. Сначала включите двухэтапную аутентификацию
  2. Затем создайте пароль приложения

Если пишет «настройка недоступна» — используйте вариант Б ниже.

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш@gmail.com
SMTP_PASS=пароль_приложения_16_символов

Вариант Б — Яндекс Почта (рекомендуется: письма реально приходят на Gmail)

  1. Создайте ящик на mail.yandex.ru (или используйте существующий)
  2. Безопасность → включите двухфакторную аутентификацию
  3. Там же → Пароли приложений → создайте для «Почта»
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=ваш@yandex.ru
SMTP_PASS=пароль_приложения
ADMIN_EMAIL=andronchik665@gmail.com

Вариант В — Демо без регистрации

Дважды щёлкните setup-ethereal.bat в папке проекта → перезапустите start.bat. После заказа ссылка на письмо появится в консоли сервера.
